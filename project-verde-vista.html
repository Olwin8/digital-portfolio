Verde Vista Resort Booking Platform

A full-stack hotel booking system designed to enhance guest experience and streamline operations

Role

Full-Stack Developer & Team Leader

Technologies

Python Flask, Shelve, Bootstrap, JavaScript, SweetAlert2

Duration

5 months

Category

Web Application, Hospitality

Project Overview

The Verde Vista Resort Singapore project aimed to develop a seamless, user-friendly online platform for customers to explore, book, and manage their stays. The website enhances the customer experience by offering an intuitive interface for browsing rooms, selecting dates, making secure payments, and providing feedback, ensuring a smooth and efficient booking process.

Platform Walkthrough

Key Features

Secure Authentication

Implemented a robust login system with password protection for account creation and booking management.

Room Browsing

Detailed descriptions and visuals of room types (standard, deluxe, suite) to aid decision-making.

Date Selection

User-friendly calendar interface for choosing check-in and check-out dates with availability checks.

Payment Integration

Secure payment gateway for seamless and safe transactions with multiple payment options.

Booking Confirmation

Instant booking confirmations with detailed summaries sent via email to guests.

Feedback System

Allows customers to share experiences and suggestions for service improvement.

Development Process

1

Requirements Analysis

Interviewed resort management to understand booking workflow issues. Reviewed competitor platforms to spot best practices and unique opportunities.

2

System Architecture

Designed a Flask-based backend with Shelve for data persistence. Created a modular frontend structure with Bootstrap for responsive layouts and SweetAlert2 for interactive user feedback.

3

Core Functionality

Added secure logins, a room availability calendar, and booking system. Created an admin dashboard for staff to handle bookings and view analytics.

4

Payment Integration

Integrated secure payment processing with validation and confirmation emails. Supported multiple payment methods while meeting PCI compliance.

5

Testing & Optimisation

Tested with staff and guests, improved performance for busy periods, and ensured accessibility (WCAG).

Key Learnings

Booking Complexity

Uncovered complex booking logic, then built a reliable algorithm factoring in check-in/out times and cleaning buffers.

Mobile-First Necessity

Found 70%+ bookings were mobile, so redesigned with touch-friendly UI and progressive enhancement.

Staff Workflow

Studied front desk workflows, creating a simple guest view and a staff version with quick-editing tools.
